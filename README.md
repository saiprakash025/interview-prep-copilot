# Interview Prep Copilot

An AI-powered tool that generates tailored interview questions from a pasted job description, then tracks your prep progress across aptitude, technical, and HR rounds.

Built to prep for Indian campus placements — paste a JD, get categorized questions generated by Gemini, mark them as answered/flagged/skipped, and track progress per company.

## Live Demo

- Frontend: https://interview-prep-copilot.vercel.app
- Backend : https://interview-prep-copilot.onrender.com

*(Note: backend is on Render's free tier, which sleeps after ~15 min of inactivity — first request may take 30–50s to wake up.)*

## Tech Stack

- **Frontend:** React + Vite, React Router, Axios
- **Backend:** Node.js, Express, MongoDB (Mongoose), JWT auth
- **AI:** Google Gemini API (`gemini-3.1-flash-lite`) for question generation
- **Deployment:** Render (backend), Vercel (frontend)
- **CI:** GitHub Actions (build checks on every push)

## Features

- JWT-based auth (register/login)
- Paste a job description → auto-generates categorized interview questions (aptitude / technical / HR) with difficulty ratings
- Mark questions as answered, flagged, or skipped
- Per-JD progress dashboard with category breakdowns
- Rate-limited AI generation endpoint to control API usage

## Running Locally

You need two terminals running at once — backend and frontend are separate processes.

### Backend

```bash
cd backend
npm install
cp .env.example .env   # fill in MONGO_URI, JWT_SECRET, GEMINI_API_KEY
npm run dev
```
Runs on `http://localhost:5000`. Verify with `curl http://localhost:5000/health`.

**Environment variables (`backend/.env`):**
| Variable | Description |
|---|---|
| `MONGO_URI` | MongoDB Atlas connection string |
| `JWT_SECRET` | Any long random string, used to sign auth tokens |
| `GEMINI_API_KEY` | Free key from [aistudio.google.com](https://aistudio.google.com) |
| `PORT` | Defaults to 5000 |

### Frontend

```bash
cd frontend
npm install
cp .env.example .env   # defaults to http://localhost:5000
npm run dev
```
Runs on `http://localhost:5173`.

### Seeding demo data (optional)

To populate a demo account without calling the Gemini API (free, instant):
```bash
cd backend
node scripts/seed.js
```
Login with `demo@interviewprep.dev` / `demo1234`.

## Deployment

- **Backend** is deployed on [Render](https://render.com) as a Web Service, root directory `backend`, with `MONGO_URI`, `JWT_SECRET`, and `GEMINI_API_KEY` set as environment variables.
- **Frontend** is deployed on [Vercel](https://vercel.com), root directory `frontend`, with `VITE_API_URL` set to the deployed backend URL.
- Both auto-deploy on every push to `main`.

## CI

GitHub Actions (`.github/workflows/ci.yml`) runs on every push/PR to `main`:
- Installs backend dependencies and syntax-checks `server.js`
- Installs frontend dependencies and runs a production build

Check the **Actions** tab on GitHub to see pipeline status.

## API Overview

| Method | Route | Description |
|---|---|---|
| POST | `/api/auth/register` | Create account |
| POST | `/api/auth/login` | Log in, returns JWT |
| POST | `/api/jd` | Save a job description |
| GET | `/api/jd` | List your saved JDs |
| POST | `/api/jd/:id/generate` | Generate questions from a JD via Gemini |
| GET | `/api/questionsets/:id` | Fetch a generated question set |
| PATCH | `/api/attempts/:questionSetId/:questionIndex` | Mark a question answered/flagged/skipped |
| GET | `/api/dashboard` | Progress summary across all question sets |
